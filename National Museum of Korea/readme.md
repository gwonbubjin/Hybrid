# 프로젝트 README

# 🏛️ 국립중앙박물관 홍보 웹앱

이 프로젝트는 **국립중앙박물관을 소개하는 반응형 홍보 웹앱**으로, GitHub Pages를 통해 배포할 수 있도록 구성된 단일 페이지(One Page) 웹사이트입니다.

본 README는 전체 코드 구조 분석을 기반으로 정리되었습니다.

---

## 📌 프로젝트 개요

* 국립중앙박물관의 **전시 소개**, **이벤트 안내**, **소장품 컬렉션**, **박물관 안내 정보** 등을 한 페이지 안에 담아
  사용자에게 직관적인 정보 탐색 경험을 제공합니다.
* **모바일 최적화**, **부드러운 스크롤**, **카테고리 필터**, **검색 기능**, **햄버거 메뉴** 등 현대적인 UI/UX 요소를 포함합니다.

---

## 📁 폴더 구조

```
museum-website/
├── index.html                 # 메인 단일 페이지 (모든 섹션 포함)
├── css/
│   └── main.css               # 전체 스타일 정의 (반응형 포함)
├── js/
│   └── main.js                # 네비게이션, 검색, 필터링, 애니메이션 기능
├── images/                    # 이미지 리소스
│   ├── logo/
│   ├── exhibitions/
│   ├── events/
│   └── collections/
└── img/                       # 추가 이미지 폴더
```

## ✨ 주요 기능

### 🔹 1. 부드러운 스크롤 네비게이션

* 메뉴 클릭 시 해당 섹션으로 자연스럽게 이동
* 사용자 경험(UX) 향상

### 🔹 2. 모바일 햄버거 메뉴

* 작은 화면에서 자동으로 햄버거 아이콘 생성
* 메뉴 열기/닫기 애니메이션 적용

### 🔹 3. 컬렉션 검색 기능

* 컬렉션 영역에서 키워드 검색 가능
* 실시간 필터링

### 🔹 4. 카테고리별 필터링

* ‘전시’, ‘소장품’, ‘행사’ 등 테마를 기준으로 구분
* 사용자가 원하는 정보만 빠르게 접근 가능

### 🔹 5. 스크롤 애니메이션

* 특정 요소가 화면에 들어올 때 자연스럽게 Fade‑in
* 사이트 전체의 시각적 완성도 강화

## 🌐 GitHub Pages 배포 시 주의사항

1. **index.html은 반드시 루트 디렉터리에 위치해야 합니다.**
2. 모든 경로는 **상대 경로(./images/… 등)**로 작성했습니다.
3. GitHub Pages는 **대소문자 구분**이 엄격하므로 파일명 확인 필수.
4. 이미지 용량을 줄이면 **로딩 속도**가 크게 향상됩니다.
5. Settings → Pages에서 브랜치 선택 후 바로 배포 가능.
